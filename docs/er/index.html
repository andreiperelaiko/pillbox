Стрелка идёт от внешнего ключа (FK) к первичному (PK). Линии только под 90°.

C4-архитектура Приложение API
Аккаунты Опекунство Лекарства Уведомления точка = FK-колонка, стрелка = PK
Как читать стрелки: линия выходит из цветной строки с FK (внешний ключ) и стрелкой входит в id (или другой PK) родительской таблицы. Например: sessions.user_id → users.id.
Две линии из user_guardians в users — это не дубль: user_id (подопечный) и guardian_id (опекун), оба ссылаются на users.
